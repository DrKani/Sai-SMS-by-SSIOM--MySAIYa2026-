## ✅ What's in Good Shape

### Firebase Config Uses Environment Variables
`src/lib/firebase.ts` correctly reads all Firebase keys from `VITE_FIREBASE_*` env vars.
No API keys are hardcoded. This is the right pattern.

### Firestore & Storage Security Rules
Both `firestore.rules` and `storage.rules` use a **deny-all-by-default** approach.
Users can only read/write their own data. Admins have broader access. This is solid.

### `useGameSound.ts` — No Audio Files Needed
The game sound hook (`src/hooks/useGameSound.ts`) uses the browser's built-in
**Web Audio API** to synthesize all sounds programmatically using oscillators.
There are no missing audio files — the hook is fully functional as-is.

### Code-Splitting / Lazy Loading
`App.tsx` lazy-loads every page with `React.lazy()` and `<Suspense>`. This means
the app only downloads the code for a page when the user visits it. Good for performance.

### PWA Setup
`manifest.json` and `sw.js` (service worker) are in place. The app can be installed
on a phone's home screen and has basic offline support.

### TypeScript Types
`src/types.ts` defines clear interfaces for `UserProfile`, `Reflection`,
`BookClubWeek`, etc. This makes the code much easier to understand and maintain.

## ⚠️ File Structure Issues

### 1. `dist/` folder is committed to git
**Problem:** The `dist/` folder contains the compiled/built version of the app. It's
automatically generated by `npm run build` and should never be stored in git.
It's listed in `.gitignore` but was committed before that rule was added.

**Fix (run once in your terminal):**
```bash
git rm -r --cached dist/
git commit -m "chore: stop tracking dist/ build output"
```
After this, `dist/` will still be generated locally but git will ignore it.

### 6. Tailwind CSS loaded via CDN instead of npm
**Problem:** `index.html` loads Tailwind from a CDN link. This works, but:
- The full Tailwind CSS file is loaded (large) — no tree-shaking of unused styles
- Breaks if the CDN is offline
- Can't customise Tailwind config (custom colours, fonts, etc.)

**Fix (medium priority):**
```bash

npx tailwindcss init -p
```
Then add Tailwind to your CSS instead of the CDN script tag.
(Note: Wait until you're more comfortable with the project before doing this.)

## ⚠️ Google OAuth — Firebase Console Step Required

The code side of Google Sign-In is complete:
- `GoogleAuthProvider` is initialized and exported from `src/lib/firebase.ts`
- The `googleProvider` object is available for sign-in calls

**What's missing:** Google OAuth must be enabled in the Firebase Console.

**Steps:**
1. Go to [Firebase Console](https://console.firebase.google.com/) → your project
2. Click **Authentication** in the left sidebar
3. Click the **Sign-in method** tab
4. Find **Google** and click the toggle to **Enable** it
5. Add your app's domain to the **Authorized domains** list
   (e.g. `sms.ssiomya.org` and `localhost`)
6. Click **Save**

After this, calling `signInWithPopup(auth, googleProvider)` in `LoginPage.tsx`
will work. If the sign-in button doesn't appear in `LoginPage.tsx` yet, you'll
also need to add the UI button that calls that function.
